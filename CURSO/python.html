🧱 1. Fundamentos Básico

Tudo começa aqui. Variáveis, tipos de dados e operações são o alicerce de qualquer programa.

1.1 — O que é Python?

Python é uma linguagem de programação de alto nível, interpretada e com tipagem dinâmica. Criada por Guido van Rossum em 1991, é hoje a linguagem mais popular do mundo segundo o índice TIOBE.

  • Interpretada: não precisa compilar — rode direto no terminal
  • Tipagem dinâmica: não precisa declarar o tipo da variável
  • Multiparadigma: suporta funcional, orientada a objetos e imperativa
  • Comunidade gigante: tem biblioteca pronta pra quase tudo

1.2 — Variáveis e Tipos

Em Python, você cria variáveis simplesmente atribuindo valores. Não existe let, var ou int x.

📦 2. Estruturas de Dados Básico

Listas, tuplas, dicionários e sets — as 4 estruturas que você vai usar em 90% do código.

2.1 — Listas

Coleção ordenada e mutável. Aceita duplicatas e qualquer tipo.

python # Criar frutas = ["maçã", "banana", "uva"] numeros = [1, 2, 3, 4, 5] misto = ["a", 1, True, 3.14] # Acessar print(frutas[0]) # "maçã" print(frutas[-1]) # "uva" print(frutas[0:2]) # ["maçã", "banana"] # Adicionar frutas.append("manga") # adiciona no final frutas.insert(1, "laranja") # insere na posição 1 frutas.extend(["kiwi", "pera"]) # junta outra lista # Remover frutas.remove("banana") # remove pelo valor frutas.pop() # remove último frutas.pop(0) # remove pelo índice del frutas[0] # remove pelo índice # Métodos úteis numeros.sort() # ordena in-place numeros.reverse() # inverte in-place print(len(numeros)) # tamanho print(sum(numeros)) # soma print(min(numeros)) # menor print(max(numeros)) # maior print(numeros.count(3)) # quantas vezes o 3 aparece print(numeros.index(3)) # posição do 3

2.2 — Tuplas

Coleção ordenada e imutável. Mais rápida que lista. Use quando o dado não deve mudar.

python coordenadas = (-23.55, -46.63) cores_rgb = (255, 128, 0) # Acesso igual à lista print(coordenadas[0]) # -23.55 # Desempacotamento (unpacking) lat, lon = coordenadas print(f"Latitude: {lat}, Longitude: {lon}") # Trocar variáveis (swap) — truque clássico a, b = 10, 20 a, b = b, a # agora a=20, b=10

2.3 — Dicionários

Coleção de chave: valor. O mais usado para representar dados estruturados (tipo JSON).

python cliente = { "nome": "Letícia", "idade": 32, "servico": "Corte + Escova", "valor": 180.00 } # Acessar print(cliente["nome"]) # "Letícia" print(cliente.get("telefone", "N/A")) # "N/A" (chave não existe) # Adicionar / atualizar cliente["telefone"] = "11999887766" cliente["valor"] = 200.00 # Remover del cliente["idade"] cliente.pop("telefone") # Iterar for chave, valor in cliente.items(): print(f"{chave}: {valor}") # Métodos úteis print(cliente.keys()) # dict_keys(['nome', 'servico', 'valor']) print(cliente.values()) # dict_values(['Letícia', 'Corte + Escova', 200.0])

2.4 — Sets (Conjuntos)

Coleção não ordenada, sem duplicatas. Perfeito pra eliminar repetições.

python servicos = {"Corte", "Escova", "Coloração", "Corte"} print(servicos) # {'Corte', 'Escova', 'Coloração'} — sem duplicata # Operações de conjunto a = {1, 2, 3} b = {2, 3, 4} print(a | b) # {1, 2, 3, 4} — união print(a & b) # {2, 3} — interseção print(a - b) # {1} — diferença # Caso prático: clientes únicos clientes_jan = {"Ana", "Bia", "Carol"} clientes_fev = {"Bia", "Carol", "Diana"} novos = clientes_fev - clientes_jan print(f"Clientes novos em fev: {novos}") # {'Diana'}

Quando usar o quê? Lista → sequência ordenada mutável. Tupla → dados que não mudam (coordenadas, configs). Dicionário → dados com nome (JSON). Set → valores únicos.

3.1 — if / elif / else

python faturamento = 15000 if faturamento >= 20000: print("🟢 Meta batida!") elif faturamento >= 15000: print("🟡 Quase lá!") else: print("🔴 Abaixo da meta") # Operador ternário (uma linha) status = "Ativo" if faturamento > 0 else "Inativo"

python status_code = 404 match status_code: case 200: print("OK") case 404: print("Não encontrado") case 500: print("Erro do servidor") case _: print("Status desconhecido")

⚡ 4. Funções Básico

Funções organizam seu código em blocos reutilizáveis. São o primeiro passo pra escrever código profissional.

4.1 — Definição e chamada

python def saudacao(nome): return f"Olá, {nome}!" print(saudacao("Róni")) # "Olá, Róni!"

4.2 — Parâmetros: default, *args, **kwargs

python # Valor padrão def calcular_desconto(valor, desconto=0.10): return valor * (1 - desconto) print(calcular_desconto(100)) # 90.0 (10% padrão) print(calcular_desconto(100, 0.20)) # 80.0 (20%) # *args — aceita N argumentos posicionais def soma(*numeros): return sum(numeros) print(soma(1, 2, 3)) # 6 print(soma(10, 20, 30, 40)) # 100 # **kwargs — aceita N argumentos nomeados def criar_cliente(**dados): for chave, valor in dados.items(): print(f"{chave}: {valor}") criar_cliente(nome="Ana", idade=25, cidade="SP")

4.3 — Lambda (funções anônimas)

python # Lambda: função de uma linha dobro = lambda x: x * 2 print(dobro(5)) # 10 # Uso prático: ordenar por critério clientes = [ {"nome": "Ana", "valor": 300}, {"nome": "Bia", "valor": 150}, {"nome": "Carol", "valor": 500}, ] # Ordenar por valor (maior → menor) clientes.sort(key=lambda c: c["valor"], reverse=True) print(clientes[0]["nome"]) # "Carol"

4.4 — map(), filter(), reduce()

python precos = [100, 200, 300, 50] # map — aplica função em cada item com_desconto = list(map(lambda p: p * 0.9, precos)) # [90.0, 180.0, 270.0, 45.0] # filter — filtra itens que satisfazem condição caros = list(filter(lambda p: p > 100, precos)) # [200, 300] # reduce — acumula valor from functools import reduce total = reduce(lambda a, b: a + b, precos) # 650

4.5 — Escopo: local vs global

python taxa = 0.10 # variável global def aplicar_taxa(valor): # 'taxa' é acessível aqui (leitura) return valor * (1 + taxa) def alterar_taxa(): global taxa # precisa declarar pra modificar taxa = 0.15

Evite usar global. É sinal de código mal organizado. Prefira passar o valor como parâmetro e retornar o resultado.

🏗️ 5. POO — Programação Orientada a Objetos Intermediário

Organize código em classes e objetos. É assim que sistemas grandes são construídos.

5.1 — Classes e objetos

python class Cliente: def __init__(self, nome, email): self.nome = nome self.email = email self.ativo = True def desativar(self): self.ativo = False print(f"{self.nome} desativado.") def __repr__(self): status = "✅" if self.ativo else "❌" return f"Cliente({self.nome}, {status})" # Usar c1 = Cliente("Ana", "ana@email.com") print(c1) # Cliente(Ana, ✅) c1.desativar() # "Ana desativado." print(c1) # Cliente(Ana, ❌)

5.2 — Herança

python class Animal: def __init__(self, nome, especie): self.nome = nome self.especie = especie def falar(self): return "..." class Cachorro(Animal): def __init__(self, nome, raca): super().__init__(nome, "Canino") self.raca = raca def falar(self): return "Au au! 🐕" class Gato(Animal): def falar(self): return "Miau! 🐱" rex = Cachorro("Rex", "Golden") mia = Gato("Mia", "Felino") print(rex.falar()) # "Au au! 🐕" print(mia.falar()) # "Miau! 🐱" print(isinstance(rex, Animal)) # True

5.3 — Propriedades (@property)

python class Produto: def __init__(self, nome, preco): self.nome = nome self._preco = preco # convenção: _ = "privado" @property def preco(self): return self._preco @preco.setter def preco(self, valor): if valor < 0: raise ValueError("Preço não pode ser negativo") self._preco = valor @property def preco_formatado(self): return f"R$ {self._preco:.2f}" p = Produto("Shampoo", 29.90) print(p.preco_formatado) # "R$ 29.90" p.preco = 35.00 # p.preco = -10 → ValueError!

5.4 — Métodos estáticos e de classe

python class Conversor: taxa_dolar = 5.20 @staticmethod def celsius_para_fahrenheit(c): return c * 9/5 + 32 @classmethod def brl_para_usd(cls, valor_brl): return valor_brl / cls.taxa_dolar print(Conversor.celsius_para_fahrenheit(30)) # 86.0 print(Conversor.brl_para_usd(1000)) # 192.31

5.5 — Dunder methods (métodos mágicos)

python class Dinheiro: def __init__(self, valor): self.valor = valor def __add__(self, outro): return Dinheiro(self.valor + outro.valor) def __str__(self): return f"R$ {self.valor:.2f}" def __gt__(self, outro): return self.valor > outro.valor a = Dinheiro(100) b = Dinheiro(250) total = a + b print(total) # R$ 350.00 print(b > a) # True

5.6 — Dataclasses (Python 3.7+)

python from dataclasses import dataclass @dataclass class Venda: cliente: str valor: float mes: str pago: bool = True v1 = Venda("Ana", 350.00, "março") v2 = Venda("Ana", 350.00, "março") print(v1) # Venda(cliente='Ana', valor=350.0, mes='março', pago=True) print(v1 == v2) # True (compara campo a campo) print(v1.cliente) # "Ana"

🎨 10. Decorators Avançado

Decorators modificam o comportamento de funções sem alterar o código delas. São usados em frameworks como Flask e Django.

10.1 — Criando um decorator

python import time def timer(func): def wrapper(*args, **kwargs): inicio = time.time() resultado = func(*args, **kwargs) fim = time.time() print(f"{func.__name__} executou em {fim - inicio:.4f}s") return resultado return wrapper @timer def processar_dados(n): total = sum(range(n)) return total processar_dados(1_000_000) # "processar_dados executou em 0.0312s"

10.2 — Decorator com parâmetros

python def repetir(vezes): def decorator(func): def wrapper(*args, **kwargs): for _ in range(vezes): resultado = func(*args, **kwargs) return resultado return wrapper return decorator @repetir(vezes=3) def gritar(msg): print(msg) gritar("Python!") # Python! # Python! # Python!

10.3 — Decorators úteis do dia a dia

python from functools import lru_cache, wraps # Cache automático (memoization) @lru_cache(maxsize=128) def fibonacci(n): if n < 2: return n return fibonacci(n - 1) + fibonacci(n - 2) print(fibonacci(100)) # instantâneo, sem cache seria lentíssimo # Log com functools.wraps (preserva nome da função) def log(func): @wraps(func) def wrapper(*args, **kwargs): print(f"[LOG] Chamando {func.__name__}") return func(*args, **kwargs) return wrapper

⚙️ 11. Generators e Iterators Avançado

Generators processam dados sob demanda (lazy evaluation). Ideais para datasets grandes que não cabem na memória.

11.1 — yield vs return

python # Generator function — usa yield def contar_ate(n): i = 1 while i <= n: yield i # retorna o valor e "pausa" i += 1 # Consumir um por um for num in contar_ate(5): print(num) # 1, 2, 3, 4, 5 # Comparação de memória # Lista: [0, 1, 2, ..., 999_999] → ocupa ~8MB na RAM # Generator: gera um por vez → ocupa ~120 bytes lista = [x ** 2 for x in range(1_000_000)] # lista na memória gen = (x ** 2 for x in range(1_000_000)) # generator (lazy)

11.2 — Caso prático: processar CSV gigante
