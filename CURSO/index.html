📈 O que é Business Intelligence Fundamento

Definição que cola na entrevista

Business Intelligence é o processo de coletar, organizar, analisar e visualizar dados para transformá-los em informações que apoiam decisões de negócio. Não é uma ferramenta específica — é um conjunto de práticas, ferramentas e cultura.

Na prática, o mercado quer alguém que pegue dados bagunçados de várias fontes (planilhas, sistemas, bancos de dados), limpe e organize esses dados, e depois monte dashboards bonitos e úteis para gestores tomarem decisões. É isso. Se você sabe fazer isso, está empregado.

"O que é ETL?" ▼

Resposta: "ETL significa Extract, Transform, Load — Extrair dados de fontes diversas, Transformar (limpar, padronizar, enriquecer) e Carregar em um destino (data warehouse, modelo de dados). No Power BI, o Power Query faz esse papel. Em projetos maiores, ferramentas como dbt, Airflow ou SSIS são usadas."

"O que é esquema estrela?" ▼

Resposta: "É a forma de modelar dados para BI: uma tabela fato central (com as métricas — vendas, transações) cercada por tabelas dimensão (cliente, produto, tempo, loja). Isso otimiza performance e torna o modelo lógico para medidas DAX."

"O que é RLS no Power BI?" ▼

Resposta: "Row-Level Security (RLS) é o controle de acesso por linha de dados. Cada usuário vê só os dados que deve ver. Exemplo: o gerente da loja SP vê só os dados de SP. É configurado no modelo de dados com DAX e atribuído a usuários no Power BI Service."

-- Exemplo de regra RLS em DAX:
[regiao] = USERPRINCIPALNAME()
"Como você lida com dados sujos?" ▼

Resposta: "Primeiro identifico os problemas: duplicatas, nulos, tipos errados, nomes inconsistentes. Depois trato no Power Query (ou Python/SQL, dependendo do volume). Minhas etapas: remover duplicatas, padronizar maiúsculas/minúsculas, tratar nulos com substituição ou remoção, corrigir tipos de dados, e validar com contagens antes x depois."

🖥️ Anatomia do Power BI Desktop

Conheça as 3 views que você usará:

Report View — Onde tudo acontece

É onde você monta os dashboards. Arrastar campos para os visuais, aplicar filtros, formatar cores e layout.

  • Canvas: área central onde ficam os visuais
  • Painel de Visuais: à direita — escolha o tipo de gráfico
  • Painel de Campos: à direita — arraste colunas/medidas
  • Painel de Filtros: defina filtros visuais, de página ou de relatório
  • Páginas: abas na parte inferior — organize por tema

Estrutura clássica de páginas: 1) Visão Geral (KPIs), 2) Detalhamento Financeiro, 3) Análise de Clientes, 4) Operacional.

Data View — Inspecionar dados

Mostra os dados tabulares carregados. Use para verificar se o ETL funcionou, criar colunas calculadas e conferir tipos.

  • Veja cada tabela como uma planilha
  • Crie colunas calculadas aqui (com DAX)
  • Confira tipos de dados (texto, número, data)
  • Verifique se o Power Query limpou corretamente

Model View — Relacionamentos

Aqui você vê e configura os relacionamentos entre tabelas. É onde se monta o esquema estrela.

  • Arraste campos entre tabelas para criar relacionamentos
  • Defina cardinalidade (1:N, N:N)
  • Configure direção do filtro cruzado
  • Oculte colunas que o usuário não precisa ver

Relacionamentos errados = medidas erradas. Sempre confira a cardinalidade. A maioria dos erros de DAX vem de modelagem ruim.

🧮 DAX — As fórmulas que você PRECISA saber 80/20

DAX (Data Analysis Expressions) é a linguagem de fórmulas do Power BI. Parece Excel, mas é mais poderosa. Você NÃO precisa saber tudo — 20 fórmulas resolvem 80% do trabalho. Aqui estão elas, em ordem de importância.

🟢 Nível 1 — Medidas Básicas (aprenda primeiro)

-- Faturamento total
Faturamento = SUM( vendas[valor] )

-- Quantidade vendida
Qtd Vendas = COUNTROWS( vendas )

-- Ticket médio
Ticket Médio = DIVIDE( [Faturamento], [Qtd Vendas], 0 )

-- Clientes únicos
Clientes Únicos = DISTINCTCOUNT( vendas[cliente_id] )

-- Faturamento médio por dia
Média Diária = AVERAGEX(
    VALUES( calendario[data] ),
    [Faturamento]
)

DIVIDE vs divisão (/): sempre use DIVIDE() em vez de barra. Ele trata divisão por zero automaticamente. Isso é pergunta de entrevista e é erro comum de iniciante.

🟡 Nível 2 — Time Intelligence (o que impressiona)

Funções de inteligência temporal são o que separa um dashboard básico de um profissional. Requisito: ter uma tabela calendário no modelo.

-- Tabela Calendário (OBRIGATÓRIA)
Calendario = CALENDARAUTO()

-- Faturamento do mês anterior
Fat Mês Anterior = CALCULATE(
    [Faturamento],
    PREVIOUSMONTH( calendario[Date] )
)

-- Variação % MoM (Month over Month)
Var % MoM = DIVIDE(
    [Faturamento] - [Fat Mês Anterior],
    [Fat Mês Anterior],
    0
)

-- Acumulado do ano (YTD - Year to Date)
Fat YTD = TOTALYTD( [Faturamento], calendario[Date] )

-- Mesmo período ano anterior (YoY)
Fat Ano Anterior = CALCULATE(
    [Faturamento],
    SAMEPERIODLASTYEAR( calendario[Date] )
)

-- Variação % Ano sobre Ano
Var % YoY = DIVIDE(
    [Faturamento] - [Fat Ano Anterior],
    [Fat Ano Anterior],
    0
)

-- Média móvel 3 meses
Média Móvel 3M = AVERAGEX(
    DATESINPERIOD( calendario[Date], MAX(calendario[Date]), -3, MONTH ),
    [Faturamento]
)

Sem tabela calendário = sem time intelligence. Sempre crie uma. Se o entrevistador perguntar "por que precisa de tabela calendário?", responda: "porque as funções de time intelligence do DAX exigem uma coluna de data contínua e marcada como tabela de datas."

🔴 Nível 3 — CALCULATE (a função mais poderosa)

CALCULATE altera o contexto de filtro. É a função mais importante do DAX — aparece em 70% das medidas avançadas.

-- Faturamento só de SP
Fat SP = CALCULATE( [Faturamento], clientes[uf] = "SP" )

-- Faturamento ignorando filtro de região (ALL)
Fat Total Geral = CALCULATE( [Faturamento], ALL( clientes[uf] ) )

-- % do total
% do Total = DIVIDE(
    [Faturamento],
    CALCULATE( [Faturamento], ALL( vendas ) ),
    0
)

-- Faturamento com filtros múltiplos
Fat Premium SP = CALCULATE(
    [Faturamento],
    clientes[uf] = "SP",
    clientes[segmento] = "Premium"
)

-- Ranking de produto (RANKX)
Rank Produto = RANKX(
    ALL( produtos[nome] ),
    [Faturamento],
    ,
    DESC,
    DENSE
)

Se você entender CALCULATE + contexto de filtro, entendeu 50% do DAX. O conceito é: cada visual do Power BI aplica filtros implícitos. CALCULATE permite ignorar, adicionar ou modificar esses filtros. Ex: um gráfico filtrado por "SP" + CALCULATE com ALL() remove esse filtro → mostra o total geral.

Power Query usa a linguagem M nos bastidores. Você não precisa escrever do zero, mas entender ajuda muito:

  Filtrado

🗃️ SQL para BI — O essencial 80/20

Você já tem o ambiente SQL completo na pasta /SQL com 118 queries interativas. Aqui vou resumir os padrões que mais caem em entrevistas de BI e que você usará no dia a dia.

Os 12 padrões SQL que resolvem 80% do trabalho de BI

-- 1. Faturamento por mês
SELECT STRFTIME('%Y-%m', data) AS mes,
       SUM(valor) AS faturamento
FROM vendas
GROUP BY mes
ORDER BY mes;
-- 2. Top 10 clientes
SELECT c.nome, SUM(v.valor) AS total
FROM vendas v
JOIN clientes c ON v.cliente_id = c.id
GROUP BY c.nome
ORDER BY total DESC
LIMIT 10;
-- 3. Variação MoM (com LAG)
WITH mensal AS (
  SELECT STRFTIME('%Y-%m', data) AS mes,
         SUM(valor) AS fat
  FROM vendas GROUP BY mes
)
SELECT mes, fat,
       LAG(fat) OVER (ORDER BY mes) AS fat_anterior,
       ROUND((fat - LAG(fat) OVER (ORDER BY mes)) * 100.0 / LAG(fat) OVER (ORDER BY mes), 1) AS var_pct
FROM mensal;
-- 4. RFM simplificado (Recência, Frequência, Monetário)
SELECT cliente_id,
  JULIANDAY('now') - JULIANDAY(MAX(data)) AS recencia_dias,
  COUNT(*) AS frequencia,
  SUM(valor) AS monetario
FROM vendas
GROUP BY cliente_id;
-- 5. % do total (Window Function)
SELECT categoria,
  SUM(valor) AS total,
  ROUND(SUM(valor) * 100.0 / SUM(SUM(valor)) OVER(), 1) AS pct
FROM vendas
GROUP BY categoria;
-- 6. DRE simplificado
SELECT
  (SELECT SUM(valor) FROM vendas) AS receita,
  (SELECT SUM(valor) FROM despesas) AS despesa,
  (SELECT SUM(valor) FROM vendas) - (SELECT SUM(valor) FROM despesas) AS lucro;

com foco em KPIs comerciais e tendências mensais.

## 🛠️ Ferramentas
- **Power BI** (modelagem e visualização)
- **Python** (geração de dados fictícios)
- **SQL** (queries de validação)

## 📋 Modelo de Dados
Esquema estrela: 1 fato (vendas) + 4 dimensões
(cliente, produto, tempo, loja)

## 📊 Dashboards
| Página | Descrição |
|--------|-----------|
| Overview | KPIs, trend mensal, top produtos |
| Financeiro | DRE, margens, despesas |
| Clientes | Segmentação, RFM, retenção |

📖 Glossário BI — Termos que você precisa conhecer

KPI — Key Performance Indicator. Métrica principal do negócio.

ETL — Extract, Transform, Load. Processo de limpeza de dados.

Data Warehouse — Banco de dados otimizado para análise (leitura).

Data Lake — Armazém de dados brutos (sem transformação).

Esquema Estrela — Modelagem com fato central + dimensões.

DAX — Data Analysis Expressions. Linguagem de fórmulas do Power BI.

Power Query (M) — Ferramenta de ETL dentro do Power BI.

RLS — Row-Level Security. Controle de acesso por linha.

Gateway — Ponte entre dados on-premises e Power BI Service.

Drill-down — Navegar de resumo para detalhe (ex: ano → mês → dia).

Slicers — Filtros visuais no dashboard (botões, dropdowns).

Tooltip — Info ao passar o mouse sobre um visual.

Bookmark — Estado salvo de filtros/visuais para navegação.

Dataflow — ETL centralizado no Power BI Service (nuvem).

Workspace — Área de trabalho compartilhada no Power BI Service.

Dataset — Conjunto de dados publicado no Power BI Service.

MoM — Month over Month. Comparação mês atual vs anterior.

YTD — Year to Date. Acumulado do ano.

YoY — Year over Year. Comparação com mesmo período do ano anterior.

Churn — Taxa de perda de clientes.

NPS — Net Promoter Score. Índice de satisfação (0-100).

CAC — Custo de Aquisição de Cliente.

LTV — Lifetime Value. Valor total que um cliente gera.

OLAP — Online Analytical Processing. Análise multidimensional.

OLTP — Online Transaction Processing. Bancos transacionais.

Staging — Área intermediária onde dados ficam antes de ir pro DW.

Self-service BI — Quando o próprio usuário cria relatórios.

Data-driven — Cultura de decisão baseada em dados.
